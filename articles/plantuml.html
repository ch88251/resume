Creating Professional Diagrams with PlantUML: A Comprehensive Guide

Introduction

PlantUML is a powerful tool that enables developers, architects, and other professionals to create professional diagrams using a simple textual syntax. In this article, we will explore how to use PlantUML to design traditional UML diagrams as well as C4 model diagrams, allowing you to communicate complex ideas visually and effectively.

Getting Started with PlantUML

Installing PlantUML

Before we dive into creating diagrams, let's first ensure PlantUML is installed on your system. PlantUML can be used as a standalone tool or integrated into various editors and IDEs. Refer to the PlantUML website for the installation instructions relevant to your operating system.

Understanding PlantUML Syntax

PlantUML diagrams are defined using a textual domain-specific language. The syntax is relatively easy to grasp and consists of various keywords to represent different diagram elements. These elements include actors, objects, classes, relationships, and more. Familiarizing yourself with the syntax is crucial to create accurate and professional diagrams.

Sequence Diagram

Sequence diagrams visualize the interactions between objects in a system over time. They are excellent for representing the flow of messages and the order of method calls during the execution of a use case or scenario.

Example Sequence Diagram


@startuml
actor User
participant "System" as sys

User -> sys: Request Data
sys -> Database: Fetch Data
Database -> sys: Return Data
sys -> User: Display Data
@enduml

Class Diagram

Class diagrams illustrate the static structure of a system by visualizing the classes, their attributes, and relationships. They are essential for understanding the organization of code and the connections between various elements.

Example Class Diagram


@startuml

skinparam linetype ortho
skinparam nodesep 200

interface ProjectItem {
  + getEstimatedTime()
  + getActualTime()
}

class Task {
  + getEstimatedTime()
  + getActualTime()
}

class Deliverable {
  + getEstimatedTime()
  + getActualTime()
}

Task .up.|> ProjectItem
Deliverable .up.|> ProjectItem
Task *-up-> " 0..* " ProjectItem

@enduml

Database ER Diagram

Database ER (Entity Relationship) diagrams illustrate the attributes and relationships of database tables.

Example Database Diagram


@startuml

skinparam linetype ortho
skinparam nodesep 100

entity "flights" as e01 {
  id : number <>
  --
  airline_id : number
  origin_airport_id : number
  destination_airport_id : number
  flight_number : varchar(20)
  departure_time : timestamp
  arrival_time : timestamp
  capacity : number
  available_seats : number
}

entity "airlines" as e02 {
  id : number <>
  --
  name : varchar(100)
  code : varchar(10)
}

entity "airports" as e03 {
  id : number <>
  --
  name : varchar(100)
  code : varchar(10)
}

entity "passengers" as e04 {
  id : number <>
  --
  first_name : varchar(40)
  last_name : varchar(40)
  email : varchar(100)
  phone : varchar(15)
}

entity "bookings" as e05 {
  id : number <>
  --
  flight_id : number
  passenger_id : number
  seat_number varchar(3)
}

e01 }o.up.|| e02
e01 }o.up.|| e03
e01 ||.right.|{ e05
e05 }|.right.|| e04

@enduml
